Object Oriented Programming in C++ Complete Interview Handbook

From fundamentals to advanced C++ internals — explained for cracking technical interviews.

A class is a user-defined data type that encapsulates data and behavior. An object is a runtime instance of a class.

Access specifiers control visibility:

  • private – accessible only inside the class
  • protected – accessible in derived classes
  • public – accessible everywhere
class Student {
private:
    int rollNo;

protected:
    string branch;

public:
    string name;

    void setRoll(int r) {
        rollNo = r;
    }

    int getRoll() {
        return rollNo;
    }
};

Interview Insight: Data members should almost always be private.

A constructor is automatically invoked when an object is created. It initializes the object state.

  • Default Constructor
  • Parameterized Constructor
  • Copy Constructor
class Demo {
    int x;
public:
    Demo() { x = 0; }
    Demo(int val) : x(val) {}
    Demo(const Demo& d) {
        x = d.x;
    }
    ~Demo() {
        // resource cleanup
    }
};

Member Initializer List is mandatory for:

  • const members
  • reference members

Interview Insight: Destructors should be virtual in polymorphic base classes.

Encapsulation

Bundling data and methods while restricting access.

Inheritance

Deriving new classes from existing ones.

Polymorphism

Ability to take many forms via virtual functions.

Abstraction

Hiding internal implementation details.

class Shape {
public:
    virtual double area() = 0; // pure virtual
};

class Rectangle : public Shape {
    double l, b;
public:
    Rectangle(double l, double b) : l(l), b(b) {}
    double area() override {
        return l * b;
    }
};

Compile-time polymorphism → Function overloading Runtime polymorphism → Virtual functions 

class Base {
public:
    virtual void show() {
        cout << "Base";
    }
};

class Derived : public Base {
public:
    void show() override {
        cout << "Derived";
    }
};

Covariant Return Types: Derived class can return a more specific type.

Interview Insight: Function calls are resolved using vtable at runtime.

  • SRP – One reason to change
  • OCP – Open for extension, closed for modification
  • LSP – Derived classes must be substitutable
  • ISP – Many specific interfaces
  • DIP – Depend on abstractions
