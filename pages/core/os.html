Operating Systems Complete Interview Guide

Master processes, memory, scheduling, deadlocks, file systems, and OS internals — explained for interviews.

An Operating System (OS) is system software that acts as an interface between the user and hardware.

Functions:
  • Process Management
  • Memory Management
  • File System
  • Device Management
  • Security & Scheduling

Interview Tip: OS is a resource manager.

A process is a program in execution. Each process has its own memory and resources.

Process States:

New → Ready → Running → Waiting → Terminated

Context Switch:

Saving the state of a running process and loading another.

Interview Q: Difference between Process and Thread?

  • FCFS
  • SJF (Preemptive & Non-preemptive)
  • Priority Scheduling
  • Round Robin
Key Metrics:

Turnaround Time, Waiting Time, Response Time

Interview Tip: SJF gives minimum average waiting time.

A deadlock occurs when processes wait indefinitely for resources.

Coffman Conditions:
  • Mutual Exclusion
  • Hold and Wait
  • No Preemption
  • Circular Wait

Prevention, Avoidance (Banker’s Algorithm), Detection

  • Paging
  • Segmentation
  • Virtual Memory
  • Page Replacement (LRU, FIFO, Optimal)
Thrashing:

System spends more time paging than executing.

  • File Allocation Methods
  • Directory Structure
  • Inodes
  • Journaling
